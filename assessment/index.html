Description

For the assessment, I initially attempted to solve it by creating 1 database with the two given tables using SQLite. I used Docker as my container and Laravel as my PHP framework, but I came across few issues:

  1. While I was able to launch my files on a local server by referring to a step by step tutorial, I came to realise that I do not fully understand the concept of how would it work if I were to embed the entire folder onto my domain.
  2. While I understood and used the SQL statements effectively, the statements were saved in a separate file, and my initial database already shows the results from the statements that I had used before, rather than a "triggered event" each time the HTML page boots up.

(Temporary...) Solution

Seeing that time was running out and the challenges prevailed after much digging across the internet, I decided to use the languages that I am more familiar with: HTML and JavaScript. The following steps was what I did:

  1. I first converted the data in Table 1 and Table 2 into a JSON file, database.json.
  2. Next, I created the skeletal structure in index.html.
  3. Then, I created a JavaScript file, table1-script.js, to ensure that the script fetches the data from Table 1 of the JSON file I created in step 1. Then, a following script that creates a row of data for each unique IndexNo.
  4. table2-script.js was the next thing I created. For this, I had to make sure to "query" the values of index of Table 1, then ensure that the calculations could be done on trigger. I also created a button next to the header so that the calculation is done by clicking on the button. (Not exactly SQL, I know...) Lastly, was to create a row of data for Alpha, Beta, and Charlie.
  5. One last code: disabling the button after the calculation is done, or else the table keeps getting repeated entries of Alpha, Beta, and Charlie.
  6. Then, I added Bootstrap with some extra tingling (styles.css) into this HTML so we don't bleed our eyes while looking at it.

Reflection

I am aware that this solution may not be what is requested nor expected, but I am grateful that I was given the opportunity to practically learn more about software- and backend development as a whole. I also realised that I should stick to frontend development... But, thank you, for taking the time to consider my application and for reading this.
